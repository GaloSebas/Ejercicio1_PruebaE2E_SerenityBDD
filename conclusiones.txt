Conclusiones:

    -El portal web de prueba presenta varios desafios interesantes que permiten tomar diferentes enfoques de automatizacion y hacer uso de diferentes librerias para conseguir las validaciones deseadas. 
    -Se ha construido el proyecto de pruebas seccionado en directorios para un facil mantenimiento de los casos de prueba y la posibilidad de un rapido escalamiento.
    -Con el objetivo de conseguir una iteracion eficiente de ejecucion se han utilizado valores randomicos para que cada prueba sea distinta y el portal web pueda ser evaluado con diferentes datos en cada iteracion, por ejemplo la seleccion de productos varia entre las iteraciones.
    -Adicional a los pasos requeridos del ejercicio se han implementado validaciones con el fin de comprobar que el flujo se ejecute con la informacion esperada y el portal funcione coherentemente, por ejemplo los productos tienen diferentes nombres, diferentes precios, en cada iteracion hay un diferente total, diferentes datos del usuario, todo el ejercicio se ha construido para que al ser ejecutado valide informacion generada randomicamente en lugar de usar informacion quemada y repetitiva.
    -El ejercicio resulta muy divertido e interesante, muy util para poner en practica los conocimientos obtenidos de automatizacion.
